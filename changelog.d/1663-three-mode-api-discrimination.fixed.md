- **`test_three_mode_api.py` had 16 tests and no discrimination** (Issue #1663). Its entire substantive assertion vocabulary was `assert result is not None` (×15) plus `hasattr(result, "U"/"M")` -- it verified that `solve()` returns an object shaped like a result, never that it solved anything. Poisoning the fixed-point iterator to return **all-NaN** for both `U` and `M` left **15 of 16 tests passing**. They now assert a plausible solution: finite `U`/`M`, non-negative density, non-constant `U`, and **mass conservation** -- under the same poison, 10 of the 16 fail (5 pass, 1 xfails). The mass check is the load-bearing one and it is not the obvious choice: measured across the three schemes this file exercises, `np.isfinite` and `M >= 0` are **true for all of them**, including the one whose mass grows by a factor of 6378 (Issue #1671, now a strict `xfail` here). `result.mass_conservation_error` could not be used, since it is never populated on this path and reads `0.0` even for that solve (Issue #1672).
