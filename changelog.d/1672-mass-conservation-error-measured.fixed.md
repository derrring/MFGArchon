- **`SolverResult.mass_conservation_error` was a constant `0.0`, not a measurement** (Issue #1672). The field is documented as `max|integral(m) - 1| over time steps` and defaulted to `0.0`, while **nothing anywhere in the package ever wrote it**. A grep for the field name finds `sinkhorn_solver.py:76` and `:265`, but those belong to `SinkhornSolverResult` -- a standalone dataclass whose MRO is `[SinkhornSolverResult, object]` and which does not subclass `SolverResult`. Same name, different class: the substitution this change exists to stop, committed while describing it. So every coupled solve reported perfect mass conservation, including the `FDM_CENTERED` case whose mass reached `6378` (Issue #1671). This is why Issue #1673 could not use the field and had to assert mass conservation by hand: it read `0.0` for the divergent solve and for a healthy one alike. The coupling loop now measures it where `M` and the geometry are both in hand, and the default becomes **`None`** -- `0.0` is a valid measurement (a solve that conserved mass exactly), so an unwritten field must not look like one. The cell measure comes from `geometry.volume_element()` rather than a second hand-rolled `dx`: the existing `calculate_total_mass(M, dx)` helper is 1-D only (`np.sum(M * dx, axis=1)`), and adding a second owner for the same quantity is the defect class this repo keeps paying for. A geometry that cannot express a volume element yields `None`, not a fabricated zero. Note for absorbing or Dirichlet walls the deviation is physical outflow rather than an error, and the docstring now says so; giving that case its own semantics is not attempted here. One caveat worth knowing when reading the number: several FP paths renormalize mass every step -- `fp_fdm.py:752` (strict-adjoint, non-default), `fp_gfdm.py:605`, `fp_network.py:363` (gated on `enforce_mass_conservation`, which defaults True) -- so on those a near-zero value is structural rather than evidence of health. The default FDM path does not, which is why an absorbing wall shows up: no-flux gives `7.5e-15`, Dirichlet-0 gives `9.5e-01` with mass draining 1.0 to 0.049.
