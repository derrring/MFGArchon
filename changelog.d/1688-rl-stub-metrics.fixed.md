- **Two `BaseRLSolver` hooks fabricated a metric nobody computed** (Issue #1688). `evaluate_nash_gap()` returned `0.0` from a body that reads nothing -- and a Nash gap of zero is the report "exact equilibrium reached", the strongest possible claim about the quantity an RL caller trusts to decide whether training converged. `scale_to_mean_field()` returned the finite-population solution unchanged while its name and docstring promised a limit; both of its branches returned the same thing, so the `population_size` test was dead. Both now raise, matching the variational / OT / neural family, which already raises for the same kind of unimplemented hook rather than returning a passing metric. The infinite-population branch of `scale_to_mean_field` is kept: the mean field limit of an infinite population is itself, so returning the solution unchanged is the right answer there rather than a missing implementation. There are **zero overrides and zero callers** today, so nothing that worked stops working -- this closes a trap before it is sprung, while the RL subtree is 39 files rather than 139.
