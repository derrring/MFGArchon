- **`SeparableLagrangian` is now self-conjugate** (Issue #1645, capability `B2` of the #1642 map). `__call__` returned `L_ctrl + V + f` while the class's own `evaluate_hamiltonian` returned `H_ctrl + V + f`; a Legendre transform flips the non-kinetic sign, so the pair was off by exactly `2(V+f)`, constant in `p`. The running cost is now `L_ctrl - V - f`, which is what `canonical_cs` and the `Dual*` pair already used and what the repo's HJB assembly (`-d_t u + H_ctrl + V + f = 0`) forces. **Behavioral for the semi-Lagrangian DPP path** (non-smooth control costs, where `_use_dpp` is True): against an HJB-FDM reference the error was `V + O(h)` -- mesh-independent and exactly linear in the potential amplitude (0.0054 / 1.0054 / 2.0054 / 5.0054 at V = 0 / 1 / 2 / 5), a wrong limit rather than discretization error. It is now `O(h)` and converges (2.0054 / 2.0018 / 2.0004 -> 0.0054 / 0.0018 / 0.0004, ratios 3.03 / 4.95). Quadratic control costs are unaffected: they are smooth, so the DPP path is never taken.
