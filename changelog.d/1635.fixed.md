- **Semi-Lagrangian per-node failures now propagate instead of substituting a stale value** (Issue #1635). Two per-point loops in `HJBSemiLagrangianSolver` caught `Exception`, logged a warning, and assigned `U_star[i] = U_next[i]` -- the value at the next time level, i.e. no update at all for that node. Because the substitution is finite by construction, the NaN/Inf guard in `solve_hjb_system` could not see it, and the solver returned a plausible, silently wrong value function with no record of which nodes were contaminated. Reached at defaults (adaptive substepping engages whenever CFL > 1, with no opt-in flag). The two loops were byte-identical copies and now share one owner, `_advect_pointwise`, which raises with the grid index, coordinate and timestep and chains the original error. Measured before changing: with counters on both handlers, the pre-change tree swallows zero times across the CI-marker-filtered suite (5130 collected, 5096 passing, 27 skipped), so this is a no-op for every run that currently succeeds.
