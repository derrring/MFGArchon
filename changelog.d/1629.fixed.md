- **Fail-fast ratchet now measures what it claims** (Issue #1629). `scripts/check_fail_fast.py` scans the AST instead of raw text: bound (`except Exception as e:`), tuple and multi-line broad handlers are now counted (11 -> 115 real sites), silent `pass` under a bound handler is counted (60 -> 95), and `hasattr` mentions inside docstrings/comments are no longer counted as calls (164 -> 129). A file that cannot be parsed now raises instead of being silently reported as clean. The baseline is regenerated to the true counts, so the ratchet fails on any *new* violation rather than only on the 9.6% of handlers the regex happened to see.
