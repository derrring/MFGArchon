- **`validate_hamiltonian_consistency` can now fail a problem instead of only warning** (Issue #1642, capability `C1`). The validator held only `add_warning`, so it could never set `is_valid` False and the `if not result.is_valid: raise` branch in `MFGProblem` was structurally dead for it -- a check that could not gate. Genuine inconsistencies now raise with the offending values named; heuristic observations remain warnings.
