- **`LagrangianBase.optimal_control` now returns the same alpha* as `HamiltonianBase.optimal_control`** (Issue #1642, capability B5). The base returned the bare conjugate maximizer `argmax_alpha {p.alpha - L}` -- i.e. `+p/lambda` under MINIMIZE -- while its own docstring promised "same alpha* as `HamiltonianBase.optimal_control()`", which returns `-p/lambda`, and while the analytic `SeparableLagrangian.optimal_control` override on the same class hierarchy also returns `-p/lambda`. Any `LagrangianBase` subclass that did not override the method therefore got a drift of the wrong sign, on a documented public subclassing path. `optimal_control` is now `-sign * dH/dp`: flipped under MINIMIZE, unchanged under MAXIMIZE (where the old code was accidentally right, since `sign = -1` cancels).
- `LagrangianBase.evaluate_hamiltonian` was **correct only because it compounded that error with a second one**: it evaluated `p.alpha* - L(alpha*)` at the wrong alpha*, and the two sign errors cancelled to the right `H`. Fixing either site alone would have shipped a regression -- under MINIMIZE the composed `DualLagrangian.evaluate_hamiltonian` would have returned `-H` for symmetric costs, and for an asymmetric `L` neither `L*(p)` nor `L*(-p)`. Both sites were fixed together and now share one owner, the new private `LagrangianBase._conjugate_argmax`, so the two consumers cannot re-fork into opposite conventions. `evaluate_hamiltonian` remains the convex conjugate `L*(p)`, sense-independent, matching `DualHamiltonian.__call__` and `ControlCostBase.evaluate` (Issue #1185).
- Measured on a harness covering every path that reaches either method (`DualLagrangian` over Quadratic/L1/Bounded x MINIMIZE/MAXIMIZE x V=f=0/V,f!=0, a plain non-overriding subclass, an asymmetric `L`, and the nD branch): **every** `evaluate_hamiltonian` value is byte-identical before and after, and the only changed values are the 34 MINIMIZE `optimal_control` rows, each an exact sign flip with the magnitude unchanged to the last printed digit. Pinning tests in `tests/unit/test_core/test_lagrangian_base_alpha_sign_1642.py` fail on a revert of either site individually and on a revert of both together.
