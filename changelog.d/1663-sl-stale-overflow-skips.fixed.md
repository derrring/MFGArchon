- **Two semi-Lagrangian property tests were skipped for a defect that no longer occurs** (Issue #1663). `test_solution_finiteness` and `test_solution_smoothness` carried `@pytest.mark.skip(reason="Semi-Lagrangian method can have numerical overflow issues with certain configurations (Issue #600)")`. Removed, and the file passes: 62 tests, no overflow. The skip reasons had outlived the defect they named.
- **The smoothness assertion was scaled to the terminal data instead of a hardcoded ceiling, and tightened** (Issue #1663). It asserted `max|diff(U)| < 100.0` against a measured `1.04e-02` -- a factor of 9626, so the solve had to degrade by four orders of magnitude to fail it. The bound is now `2x` the terminal condition's own roughness. The healthy ratio is stable at `1.03-1.09` across `Nx` 31 to 101, so this leaves ~1.85x headroom, and it fires: injecting a sawtooth trips the new bound at amplitude ~0.05 against ~50 for the old one, and a linear-to-nearest interpolation regression (`hjb_semi_lagrangian.py:1149`) reaches `2.76` on this fixture and is caught. That catch is grid-dependent -- the same regression gives `1.11` at `Nx=31` and `1.76` at `Nx=81` -- so it is one regression this bound happens to catch, not a general interpolation guard.
- **Disclosure: both tests remain weak, in ways worth naming precisely** (Issue #1663). The *smoothness* fixture exercises little of the scheme: `sigma = 0.0`, so the Crank-Nicolson diffusion step is a no-op; no function in `hjb_sl_interpolation.py` is called at all (established with `sys.settrace`, not by monkeypatching -- `from X import f` binds in the importing module, so patching the defining module intercepts nothing); `reflect_into_domain` is called 50 times but is bit-for-bit inert here; and the terminal data `0.5(x-0.5)^2` is exactly symmetric about the domain midpoint, leaving the test structurally blind to direction-reversal bugs. Zeroing `_compute_gradient` moves the answer only ~6%. Several severe mutations pass both tests: zeroing the terminal condition, running the time loop forward, leaving the last timestep unsolved, dropping the running cost, and `dt` x10. The *finiteness* fixture is a different code path -- CFL 6.26, so it substeps through `_solve_timestep_semi_lagrangian_with_dt` -- and does catch characteristic-velocity mutations (negation, zeroing) and a cost-sign flip. Its own `assert np.all(np.isfinite(...))` is close to unreachable, since the solver raises `ValueError` on non-finite values first; it catches overflow via that raise, not via its assertion. Making the smoothness test genuinely discriminating needs a fixture with non-zero diffusion, off-grid departure points, and an asymmetric terminal condition -- a new test, not stricter numbers on this one. Filed as a defect class on #1663.
