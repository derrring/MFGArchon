The fixture is insensitive to the solver's own machinery, which was measured rather than assumed. `sigma = 0.0`, so the Crank-Nicolson diffusion step is a no-op; no function in `hjb_sl_interpolation.py` is called at all during the solve (verified by profiler, not by patching -- `from X import f` binds in the importing module, so patching `X.f` intercepts nothing here); and forcing `_compute_gradient` to return zeros changes the answer by only about 6% (roughness `1.04e-02` to `9.80e-03`, transport `1.060` to `1.000`), which no aggregate assertion can separate from a healthy solve without becoming flaky. Making these tests discriminating needs a fixture with non-zero diffusion and departure points that leave the grid, not stricter numbers on this one.
