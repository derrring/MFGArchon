- **The boundary-condition guide stated the HJB form of `NO_FLUX` as if it applied to both equations** (Issue #1686). `docs/user/guides/boundary_conditions.md` gave one table row per BC type, written in terms of a generic `u`, so `NO_FLUX` read as `du/dn = 0`. On the FP side that is the wrong condition: no-flux there means zero *total* flux, `J.n = 0` with `J = v*m - D*grad(m)`, which with drift at the wall implies `D dm/dn = (v.n) m` and hence a **non-zero** normal derivative. The numerics were never wrong -- `fp_fdm_alg_divergence_centered.py:273` enforces the boundary flux as exactly zero, which is what conserves mass -- only the documentation was, and it is the reason Issue #1686 was filed as blocked on "an undeclared convention". The table is now split by equation, the FP row states the flux form, and the deprecated `NoFluxCalculator` (an alias for `ZeroGradientCalculator` since v0.16.11) is distinguished from `ZeroFluxCalculator`. `no_flux_bc()`'s docstring said "zero normal derivative ... common for Fokker-Planck equations", which is the one combination that is wrong; it now states both sides.
