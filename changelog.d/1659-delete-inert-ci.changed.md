- **Deleted the CI surface that could not fail** (Issue #1659). A green checks list was reporting jobs that never ran or could not report a failure: `pr_mypy_expansion.yml` (**50/50 runs skipped**, never once executed, gated on a branch-name filter); `performance_regression.yml` entirely -- its `benchmark` job never executed a benchmark (`pytest: error: argument --benchmark-json: can't open '.benchmarks/pr_results.json'`, exit 4, three `continue-on-error: true`, **12/12 runs reported PASS**) and its `performance-tracker` job measured `np.random.seed(42)` against a `# PLACEHOLDER: Simulate solver execution` that never calls a solver; `ci.yml`'s `CI/CD Summary`, which `needs` three jobs, references none of their results, and printed `- ✅ Type Safety: Mypy type checking` unconditionally -- observed on the v0.21.0 release run alongside `failure Test Suite`; and two duplicate docstring walks (`ci.yml`, `modern_quality.yml`) that both print warnings and `exit 0`. Also removed four hardcoded echoes asserting `366 → 0 errors (100% achieved)`, a measurement the `mypy ... || true` above them does not take and which is false (the real whole-package count is in the thousands).
- **Fixed the release cascade that had never built a wheel** (Issue #1659). `integration-tests` and `build-validation` were chained behind `test-suite`, which fails on release, so **8 of 8 releases** back to v0.19.7 skipped both -- the wheel build, `twine check` and install-from-wheel smoke have never guarded a release. Both now depend on `import-validation` only: packaging and the numerical suite answer different questions and must not be chained.
- **Renamed `Test Suite` → `Smoke subset (NOT the full suite)`** in `ci.yml`. Since #1656 the PR path runs `test_core` + `test_config` only, so the old name made a green check read as "your tests passed", which is false.
- Coverage upload is now gated on `github.event_name == 'release'`; `--cov-report=xml` is emitted only by the release step, so on PRs it found 0 files and passed. The parameter-migration check lost its `|| true` and now asserts the registry is non-empty -- as written it verified that the runner had a Python interpreter. `deprecation-check.yml` no longer triggers on a `develop` branch that does not exist.
- `scripts/local_ci.sh` now validates that every `.github/workflows/*.yml` parses. Added after a malformed edit to `ci.yml` passed the gate.
